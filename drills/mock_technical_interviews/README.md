# Mock Technical Interviews

Simulações de entrevista completa, misturando SQL, Python, Pandas, modelagem, dbt e arquitetura numa única sessão cronometrada - o objetivo é praticar sob pressão de tempo, não só resolver exercícios isolados.

## Estrutura

```
mock_technical_interviews/
├── junior/mock_01.md   → 45-60 min
├── pleno/mock_01.md     → 60-75 min
└── senior/mock_01.md    → 75-90 min
```

## Como usar

1. Cronometre de verdade (use um timer) - parte do treino é responder sob pressão de tempo real
2. Responda por escrito ou em voz alta antes de checar qualquer módulo de referência
3. Os "Pontos-chave esperados" no final servem pra autoavaliação, não são gabarito literal
4. Para respostas modelo completas dos cases de arquitetura, consulte `data_system_design_drill/` (linkado em cada mock)
5. Para o gabarito dos exercícios de Python/SQL isolados, consulte os módulos correspondentes (`python_drill/`, `sql-drill/`, `data_engineering_interview_drill/`)

Esse módulo não duplica conteúdo dos outros - ele **remixa** exercícios e cases já existentes numa simulação de sessão única, testando sua capacidade de trocar de contexto rapidamente (que é uma habilidade real cobrada em entrevista).
