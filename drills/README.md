# Drills - Prática Paralela

Módulos de prática recorrente, independentes da sequência numerada da trilha principal (`../projects/`). Aqui o foco é preparação para entrevista técnica: SQL, Python, pandas, cenários de engenharia de dados, arquitetura e simulações completas.

| Pasta | O que é |
|---|---|
| [`sql-drill/`](sql-drill/) | Exercícios diários de SQL, cada drill isolado em seu próprio schema no Postgres |
| [`pandas_drill/`](pandas_drill/) | Exercícios de análise de dados com pandas, consumindo os schemas dos SQL Drills |
| [`python_drill/`](python_drill/) | Desafios de algoritmos e estruturas de dados no formato de entrevista técnica |
| [`linux_drill/`](linux_drill/) | Prática de terminal — navegação, processamento de texto, processos, diagnóstico de sistema |
| [`git_drill/`](git_drill/) | Controle de versão — mesma operação em profundidade crescente (conflito, rebase, bisect, estratégia de branching) |
| [`data_engineering_interview_drill/`](data_engineering_interview_drill/) | Cenários práticos de pipeline (dedup, CDC, watermark, checkpoint) |
| [`data_system_design_drill/`](data_system_design_drill/) | Estudos de caso de arquitetura, com resposta modelo didática |
| [`mock_technical_interviews/`](mock_technical_interviews/) | Simulações de entrevista completa, cronometradas, misturando todos os temas |

Cada pasta tem sua própria estrutura documentada no README local.
